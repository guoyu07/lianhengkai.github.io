PHP之无限极分类

  • PHP
PHP

无限极分类

无限极: 不知道分类到底有多少层(无限: 层次不确定)
无限极分类: 通过顶级分类,下面显示的一定是其子分类,子分类下显示一定又是自己的子分类

无限极分类的数据库表设计思想: 在每一个分类的后面增加一个字段,指向其父分类.

无限极分类原理

  • 给定一个需要查找顶级父分类
  • 从所有的分类中去寻找父分类 id 等于顶级分类 id 的分类
    a) 如果找到: 有子分类: 子分类还有可能有子分类: 暂停寻找当前分类的子分类, 变成寻找子分类的子分类: 直到子分类找出所有的孙子分类后,回到当前分类的子分类查找: 继续查询
    b) 如果找不到: 整个找完也没有合适: 说明没有子分类(结束)

虽然在数据表中可以通过 c_parent_id 找出一个分类的所有子分类,也可以找出子分类对应父分类. SQL 语句没有办法将所有的分类进行无限极分类.
所以,只能利用 SQL 语句查出所有的商品分类: 然后在 PHP 中对商品分类进行无限极分类.

代码实现无限极分类

1.在商品分类中对查询出来的结果进行无限极分类

2.查询出所有的顶级分类.

3.简单证明方法有效: 在获取到所有的商品分类结果之后,调用商品分类无限极方法进行处理

4.无限极分类: 找顶级分类与找顶级分类下的子分类,以及找任何一个分类的子分类的过程都是一样: 从所有的商品分类中去匹配 c_parent_id = 当前要找的商品分类的id.

5.递归实现: 递归点: 一旦当前找出来的商品分类复合匹配的条件(是某个分类的子分类) ,该子分类同样有可能是其他分类的父分类: 应该尝试找出所有的子分类.

6.递归出口: 找不到子分类,导致数据遍历结束

7.通过函数是第几层调用来区分商品分类是第几层子分类. 然后在每层找出来的子分类里面增加一个额外的属性: 用来标识当前商品分类属于第几层.
